How It Works Under the Hood

The Dispatch Flow

// Task chain step activates → AgentBoard dispatches:

1. Read agent config → runtime: "z-ai", model: "z-ai-research-v2"
2. Build prompt → system_prompt + task context + previous step output
3. Resolve MCP permissions → step.mode = "analyze" → github:read, jira:read
4. Call runtime adapter → POST z-ai API with prompt + tools
5. Collect response → store as step.output
6. Advance chain → move task to WAITING, activate next step
Key: The same Developer Agent can be used in Step 2 (verify, read-only MCP) and Step 4 (develop, write MCP) of the same chain. The agent's identity and runtime stay the same — only the step permissions change.
