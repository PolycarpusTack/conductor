AgentBoard — Enhanced Agent Creation & Task Chains

Design proposal: Reusable agents + task-level workflow chains with handoffs

Core Concept

Two Separate Concerns, Connected at Task Time

🤖

Agents = Reusable Identities

Created once, used across many tasks

An agent has a name, role, MCP connections, and general capabilities. It's like hiring a team member — they exist in your project and can be assigned to any task.

Support Analyst Developer Product Analyst Documentation Writer
🔗

Tasks = Workflow Chains

Permissions & handoffs defined per-step

Each task can have a chain of steps. Each step assigns an agent (or human) with a specific mode/permission. When one step finishes, the task moves to WAITING for the next assignee.

analyze verify develop human-review

// New model
model TaskStep {
  id String
  taskId String
  order Int
  agentId String?
  humanLabel String?
  mode String // analyze|verify|develop|human
  status String // pending|active|done
  output String?
}

Your Framework Stages Mapped

✓ Stage 1 — Identity fields on Agent (role, mission, capabilities)
✓ Stage 3 — MCP connections on Agent, mode/permissions on TaskStep
✓ Stage 4 — Orchestration = the chain itself (directed workflow)
◐ Stage 5 — Human review gates built into chain (manual CI)
○ Stage 2 — ETL/RAG (future — not in scope)
○ Stage 6 — Live observability (future — your dashboard design)
Key insight: The chain builder IS your orchestration engine. Human review steps ARE your deployment gates. No separate CI/CD system needed — it's built into the task flow.
