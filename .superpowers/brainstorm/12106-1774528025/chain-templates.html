How It All Connects

The Full Picture

CREATE AGENT (once)
→ Identity: name, role, capabilities
→ Runtime: provider + model + system prompt
→ Connections: MCPs (optional)
↓
CREATE CHAIN TEMPLATE (once per workflow type)
→ Define steps: who does what, in what order
→ Set modes: analyze, verify, develop, human
→ Toggle auto-continue per step
↓
CREATE TASK (every time)
→ Pick a template (or build custom chain)
→ Steps pre-populate → adjust if needed
→ Or just assign a single agent (no chain)
↓
EXECUTION (automatic)
→ AgentBoard dispatches to runtime (Anthropic, webhook, etc.)
→ Agent works → output saved → chain advances
→ Human steps pause in WAITING column
→ Auto steps dispatch immediately
→ Failed steps halt chain → human retries/skips
